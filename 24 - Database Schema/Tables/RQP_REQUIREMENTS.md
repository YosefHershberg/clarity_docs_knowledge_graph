# RQP_REQUIREMENTS

> [!info] Description
> This table stores the basic attributes of a requirement.

## Columns

| Column | Type | Null | Comment |
| --- | --- | --- | --- |
| `ID` | NUMERIC | NOT NULL | This column refers to the unique ID of the requirement generated by a sequence counter. |
| `TITLE` | NVARCHAR | NOT NULL | This column refers to the title of the requirement. |
| `CODE` | NVARCHAR | NOT NULL | This column refers to the unique code identifying the requirement. |
| `TYPE` | NVARCHAR | NULL | This column refers to the requirement's type. |
| `THEME` | NVARCHAR | NULL | This column refers to the theme of the requirement. |
| `DESCRIPTION` | NVARCHAR | NULL | This column refers to the requirement's description. |
| `MANAGER_ID` | NUMERIC | NULL | This column refers to the ID of the manager of the requirement. |
| `STATUS` | NVARCHAR | NULL | This column refers to the status of the requirement. |
| `COMMITTED` | NUMERIC | NULL | This column indicates whether a commitment has been made to implement this requirement: - 1 = yes - 0 = no |
| `RISK` | NVARCHAR | NULL | This column refers to the risk of the requirement. |
| `REQUESTED_BY` | NVARCHAR | NULL | This column refers to the ID of the party (e.g. customer, sales, etc) who requested that this requirement be added. |
| `RELEASE_ID` | NUMERIC | NULL | This column refers to the ID of the release in which the requirement is to be implemented. |
| `TASK_ID` | NUMERIC | NULL | This column refers to the ID of the task associated with the requirement. |
| `PROJECT_ID` | NUMERIC | NULL | This column refers to the ID of the project associated with the requirement. |
| `INVESTMENT_ID` | NUMERIC | NULL | This column refers to the ID of the investment with which the requirement is affiliated. |
| `EST_OP_COST` | NUMERIC | NULL | This column refers to the estimated operating cost for the requirement. |
| `EST_CAP_COST` | NUMERIC | NULL | This column refers to the estimated capital cost for the requirement. |
| `EST_EFFORT` | NUMERIC | NULL | This column refers to the estimated effort, in hours or days, needed to implement the requirement. |
| `EST_SIZE` | NUMERIC | NULL | This column refers to the estimated size in hours or days of the effort needed to implement the requirement. |
| `EST_HLM_SIZE` | NUMERIC | NULL | This column refers to the estimated size (high, medium, low) of the effort needed to implement the requirement. |
| `BGT_OP_COST` | NUMERIC | NULL | The budgeted operating cost for the requirement. |
| `BGT_CAP_COST` | NUMERIC | NULL | This column refers to the budgeted capital cost for the requirement. |
| `BGT_EFFORT` | NUMERIC | NULL | This column refers to the budgeted effort, in hours or days, needed to implement the requirement. |
| `BGT_SIZE` | NUMERIC | NULL | This column refers to the budgeted hours or days for the effort to implement the requirement. |
| `CURRENCY_CODE` | NVARCHAR | NULL | This column refers to the code of the currency used in the money fields of the requirement. |
| `ATTACHMENT` | NUMERIC | NULL | This column refers to the foreign key to attachments. |
| `PRIORITY_1` | NUMERIC | NULL | This column refers to the requirement's priority rating. |
| `PRIORITY_2` | NUMERIC | NULL | This column refers to the requirement's priority rating. |
| `PRIORITY_3` | NUMERIC | NULL | This column refers to the requirement's priority rating. |
| `PRIORITY_4` | NUMERIC | NULL | This column refers to the requirement's priority rating. |
| `FEATURE_OVERVIEW` | NVARCHAR | NULL | This column refers to the overview of the requirement. |
| `FEATURE_DRIVER` | NVARCHAR | NULL | This column refers to the description of the drivers for the requirement. |
| `FEATURE_VISION` | NVARCHAR | NULL | This column refers to the vision for the requirement. |
| `REQ_IMPL_PLAN` | NVARCHAR | NULL | This column refers to the implementation plan for the requirement. |
| `REQ_IMPACT_ANALYSIS` | NVARCHAR | NULL | This column refers to the impact analysis for the requirement. |
| `REQ_COMMENTS` | NVARCHAR | NULL | This column refers to the comments about the requirement. |
| `STORY_TEXT_1` | NVARCHAR | NULL | Text entry for part 1 of a user story for the requirement. |
| `STORY_TEXT_2` | NVARCHAR | NULL | This column refers to part 2 of a user story for the requirement. |
| `STORY_TEXT_3` | NVARCHAR | NULL | This column refers to part 3 of a user story for the requirement. |
| `APPROVEDBY_ID` | NUMERIC | NULL | This column refers to the D of the user who approved the requirement. |
| `APPROVEDTIME` | DATE | NULL |  |
| `CREATED_DATE` | DATE | NOT NULL | This column refers to the date and time this requirement was created. |
| `CREATED_BY` | NUMERIC | NOT NULL | This column refers to the resource ([[CMN_SEC_USERS]].ID) that created this row. |
| `LAST_UPDATED_DATE` | DATE | NOT NULL | The timestamp of when the record was last modified. |
| `LAST_UPDATED_BY` | NUMERIC | NOT NULL | This column refers to the ID of the resource who last modified this requirement. |
