# CMS_CONTENT_ADDINS

> [!info] Description
> This table contains the list of add-ins and content packages that are installed.

## Subject areas (ERDs)
- [[_MOC - Content Add-ins ERD]]

## Columns

| Column | Type | Null | Comment |
| --- | --- | --- | --- |
| `ID` | NUMERIC | NOT NULL | The unique identifier for the content add-in. This column refers to the primary key and is an interal ID generated by the system. |
| `CODE` | NVARCHAR | NOT NULL | The unique identifier of the content add-in as specified in the imported content package or the installed add-in. |
| `NAME` | NVARCHAR | NOT NULL | The content add-in name. |
| `DESCRIPTION` | NVARCHAR | NULL | The description of the content add-in. |
| `UPLOADED_FILE` | NUMERIC | NULL | The content add-in .jar file name. |
| `CONTENT_ADDIN_VERSION` | NVARCHAR | NOT NULL | The version number for the content add-in. Use the following format: [0-9][0-9].[0-9].[0-9] Example: 12.3.4 |
| `STATUS` | NVARCHAR | NOT NULL | The status of the content add-in. Values: NOT_STARTED, PROCESSING, INSTALLED, INSTALLATION_FAILED, UPGRADE_READY |
| `IS_LEGACY` | NUMERIC | NOT NULL | Indicates whether add-in is a legacy add-in or a content package add-in. Values: 0=Content packages add-in, 1=Existing add-in (for example, csk) |
| `IS_MIGRATION_PACKAGE` | INTEGER | NOT NULL | Indicates whether the add-in is a migration package (that is, the add-in was installed in migration mode). Values: 0=Not a migration package, 1=Migration package |
| `CREATED_DATE` | DATE | NOT NULL | The date the content package was created. |
| `CREATED_BY` | NUMERIC | NOT NULL | The user who created the content add-in package. |
| `LAST_UPDATED_DATE` | DATE | NOT NULL | The date of the last modification to the content add-in. |
| `LAST_UPDATED_BY` | NUMERIC | NOT NULL | The user who last updated the content add-in. |
| `EOL_DATE` | DATE | NULL |  |
