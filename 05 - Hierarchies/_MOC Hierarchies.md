# Hierarchies — Map of Content

**Hierarchies** in Clarity PPM provide a powerful multi-level investment analysis framework. A hierarchy organizes investments (projects, custom investments, ideas) into a tree structure — typically program → project → sub-project — and allows managers to analyze aggregated financials, tasks, risks, and status across the entire tree in a single view.

> [!info] Hierarchies vs. the Hierarchy Module
> - **Hierarchies (this section)** — A standalone Clarity feature for creating and analyzing multi-level investment trees.
> - **[[Hierarchy Module]]** — A tab *within* an individual investment that shows its direct children.
> Both features can work together, but the standalone Hierarchies section is far more powerful for analysis.

## Key Concepts at a Glance

| Term | Description |
|------|-------------|
| **Hierarchy** | A named multi-level tree of related investments |
| **Hierarchy Node** | An investment at any level in the tree (root, parent, child, leaf) |
| **Root** | The top-level investment in the hierarchy (typically a Program or Portfolio) |
| **Rollup** | Aggregation of data (cost, effort, % complete) from children up to parents |
| **Cross-Object Grid** | A special grid that shows attributes from multiple investment types in one table |
| **Objectives Hierarchy** | A hierarchy structure that maps investments to OKRs (Objectives and Key Results) |
| **Hierarchy Blueprint** | Admin-configured template controlling which fields and modules appear in a hierarchy |
