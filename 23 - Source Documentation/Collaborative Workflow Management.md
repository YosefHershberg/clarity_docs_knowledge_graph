Collaborative Workflow Management (CWM) creates a centralized place for creating, editing, and tracking Checklists and To Dos. You can also share the checklists with other Clarity users. CWM promotes work visibility for all team members and stakeholders. It keeps everyone well-informed.

- Common Search Component

## Introduction

One of the biggest challenges for Investment managers is setting a process that accommodates ad-hoc activities to minimize disturbance and interruption for your team. Use Collaborative Workflow Management to include unplanned tasks into your workflow and support all those ad-hoc activities in your sprint.

Collaborative Workflow Management helps to:

1. **Plan For Ad-hoc Activities**: You can define ad-hoc activities to be done and collaborate with other Clarity users. Ensure your team progresses on primary tasks while managing ad-hoc activities. You can create the Checklists with a list of To Dos in My Workspace.
2. **Prioritize Ad-hoc Activities**: You can identify the ad-hoc activities that are important to include in your sprint. You can set the priority level by sorting the To Dos in checklists. If there is a dependency to complete an activity, you can block To Dos and add a description to it.
3. **Allocate Resources for Ad-hoc Activities**: You can assign an Owner and set a due date for To Dos after setting the priority list. Once the To Do is complete, you can mark it as completed.
4. **Track Ad-hoc Activities**: You can share the checklists with Stakeholders so that they can view the completed To Dos and overdue activities. You can also create customized smartlists based on the conditions. My Workspace makes work visible to stakeholders and helps to track the progress of their ad-hoc activities.
5. **Centralize Ad-hoc Activities**: You can see all To Dos associated with Investments, Tasks, Risks, Issues, Changes, and Checklists where you are assigned as Owner and To Dos created by you.

## Overview of My Workspace

My Workspace has two main sections: **Checklists** and **Smartlists**.

### Checklists

A checklist is a container that allows you to manage To Dos without associating it with investments. You can view the following checklists in the Checklist Panel:
- Checklists associated with investments
- Checklists created in My Workspace

You can perform the following actions in Checklists:
- Create a checklist
- Set a due date for the checklist
- Rename/Copy/Delete the Checklist
- Share the checklist with other team members as Collaborators

After creating the checklist, you can:
- Create To Dos
- Assign a resource and set a due date for To Dos
- Rearrange To Dos
- Block or delete To Dos in the Checklist
- Mark To Dos as Completed

### Smartlists

A Smartlist is a filter. It allows you to configure a list that displays To Dos across the following areas in Clarity:
- Investments
- Tasks
- Risks, Issues, Changes
- Checklists

Clarity provides a "My To Dos" smartlist for every user. It displays all To Dos across Investments, Tasks, Risks, Issues, Changes, and Checklist for the current user. The "My To Dos" smartlist cannot be deleted, but the current user can modify the condition and rename the "My To Dos" smartlist.

You can perform the following actions in Smartlists:
- Create a smartlist based on conditions
- Edit the smartlist
- Delete the smartlist

After creating a Smartlist, you can:
- Sort To Dos based on Name, Date, and Owner
- Block or unblock To Dos in Smartlist
- Mark To Dos as Completed

> [!NOTE]
> You can mark a completed To Do as Not Completed. So it appears in your Due list again.

## Pre-requisites

> [!IMPORTANT]
> The following table explains the impact on various user actions on checklists:

| Sl. No. | Actions | What happens in My Workspace? |
|---------|---------|-------------------------------|
| 01 | When you make an investment as a template or inactive | Checklists associated with the Investment will NOT be visible in My Workspace |
| 02 | When checklists are available in a template, and if you create an investment using the template | Checklists associated with the Template will be visible in My Workspace |
| 03 | When you convert an idea to a project and if checklists are available in the idea | Checklists associated with the Idea will be available in My Workspace |
| 04 | When you convert an idea that has checklists to a project and you choose a template with checklists | Checklists associated with Idea and Template will be available in My Workspace |

## Access Rights to Work with My Workspace

Ensure that you have the following access rights to use this feature:
- **My Workspace - Navigate**: Allows user access to my workspace pages in the Clarity. The user will be able to access the checklists to which the user has access.
- **Checklist - Create**: Allows user to create Checklists.

You need to have the following access rights to work with To Dos in My Workspace:
- **To Do - Create - All**: Allows user to create To Dos in the Clarity for which the user has access to view associated objects.
- **To Do - Delete - All**: Allows user to delete all To Dos in the Clarity for which the user has access to view To Dos and to view associated objects.
- **To Do - Edit - All**: Allows user to view and edit all To Dos in the Clarity for which the user has access to view associated objects.
- **To Do - View - All**: Allows user to view all To Dos in the Clarity for which the user has access to view associated objects.

## Managing Checklists

You can see the list of all of the checklists for which you are the owner or collaborator. You can also see the checklists created for investments (Projects, Ideas, and Custom Investments) in the list.

You can perform the following actions in Checklist:
- Creating a Checklist
- Setting Due Date for a Checklist
- Copying a Checklist
- Renaming a Checklist
- Changing the Status of a Checklist
- Moving a Checklist to an Investment/Agreement
- Removing a Checklist from an Investment/Agreement
- Deleting a Checklist
- Transferring Checklist Ownership

### Creating a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In **Checklists**, click **+** to add a new Checklist.
3. Enter a name for the checklist.
4. Click **Create**.

> [!NOTE]
> The Maximum Size of the Checklist Name is set to 256 characters.

### Setting Due Date for a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to set a due date.
3. Click **Three-dot Action Menu** and select **Set Due Date**.
4. Select a date from the calendar.

### Copying a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to copy.
3. Click **Three-dot Action Menu** and select **Copy**.
4. Enter the name of the Checklist and click **Copy**.

> [!IMPORTANT]
> When you copy a checklist, the following items are duplicated in the copied checklist:
> - Due date of the Checklist
> - All the To Dos in the Checklist along with:
>   - Owner of the To Do
>   - Due Date of the To Do
>   - Link attached to the To Do
> When you copy a checklist, the following changes happen in the copied checklist:
> - Removes collaborator(s) from the checklist
> - Changes the state of the Blocked To Do to Unblocked To Do
> - Removes Blocked Description of the To Do
> - Changes the state of the Completed To Do to Not Completed To Do
> - Removes attachment(s) from the checklist

### Renaming a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to rename.
3. Click **Three-dot Action Menu** and select **Rename**.
4. Rename the Checklist and click **Save**.

### Changing the Status of a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to change the status.
3. Click **Three-dot Action Menu** and select **Mark Complete**.

You can see the Checklist under the **Completed** heading in the Checklist Panel. The Completed heading in the Checklist panel will be collapsed by default, and you can expand it to see the completed checklists.

> [!NOTE]
> Clarity enables the **Mark Open** option for the completed Checklist. You can re-open the checklist using **Mark Open** in **Three-dot Action Menu**.

### Moving a Checklist to an Investment/Agreement

You can move the checklist from one Investment/Agreement to another Investment/Agreement using **Move Checklist** in **Three-dot Action Menu**.

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to move from one Investment/Agreement to another Investment/Agreement.
3. Click **Three-dot Action Menu** and select **Move Checklist**.
4. Select an Investment/Agreement where the checklist needs to be moved.
5. Select the specific Investment/Agreement in the **Move Checklist to (Investment/Agreement)** pop-up window displaying object-specific list.
6. Click **Move**.

The Checklist will be moved to the selected Investment/Agreement. You can see a link to the Investment/Agreement in the Checklist.

> [!NOTE]
> Some of the key points to remember are:
> 1. If you don't associate a checklist with any investment/agreement, then you will have only the option to move the checklist to an existing investment or agreement. You don't see the **Remove Checklist** option in the **Three-dot Action Menu** for this checklist.
> 2. If you move a checklist to a template or inactive investment, then the checklist will disappear from My Workspace.

### Removing a Checklist from an Investment/Agreement

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to remove from Investment/Agreement.
3. Click **Three-dot Action Menu** and select **Remove Checklist from Investment/Agreement**.

The Checklist will be removed from the Investment/Agreement. The link to the Investment/Agreement will be removed from the Checklist.

### Deleting a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to delete.
3. Click **Three-dot Action Menu** and select **Delete**.
   The Delete Checklist pop-up window opens with the message "All To Dos in this checklist will be deleted".
4. Click **Delete**.

> [!NOTE]
> If you delete the checklist from "My Workspace", then it is also deleted from the Investment/Agreement.

### Transferring Checklist Ownership

**Co-Owner** has all the rights of **Owner**, including the ability to delete the checklist and remove other collaborators from the checklist.

> [!IMPORTANT]
> Owner cannot be removed. If you move a checklist to an investment, then the manager of the investment is added as a Co-Owner of the checklist.

## Managing Sections

You can efficiently manage your To Dos by grouping them into sections. You can create new sections, customize section names to suit your organizational preferences, easily reorder sections to prioritize tasks and remove sections as needed. The section provides a seamless and intuitive way to organize and streamline your To Dos.

You can perform the following actions in section:
- Creating a Section
- Creating a To Do from Section
- Renaming the Section
- Reordering the Section
- Deleting the Section

### Creating a Section

Follow these steps:

1. In the checklist, click **+Section**.
2. Enter the name of the section and press **Enter** to add one more Section.

You can drag the existing To Dos and drop them under the new section.

### Creating a To Do from Section

Follow these steps:

1. In the section, click the **Three-dot action menu** and select **Insert To Do**.
2. Type the name of the To Do and press **Enter** to add one more To Do.

> [!NOTE]
> Some of the key points to remember are:
> - When you mark a To Do as complete, then the To Do moves under the Completed tab along with the section.
> - The Completed section does not support drag-and-drop functionality to move To Dos.

### Renaming the Section

Follow these steps:

1. In the checklist, click the section name.
2. Edit the section name and press **Enter**.

### Reordering the Section

In the checklist, drag a section and drop it at the desired place.

> [!NOTE]
> You can:
> - Drag and drop a Section
> - Drag and drop a To Do from Open state into a Section
> - Drag and drop a To Do from Section to Open state
> - Drag and drop a To Do within a Section
> - Drag and drop a To Do between different Sections
> You cannot:
> - Drag and drop a To Do or Section in Completed state
> - Drag and drop a Section within another Section
> - Drag and drop a To Do immediately below a Section

### Delete the Section

Follow these steps:

1. Select the section you want to delete in the Checklist.
2. Click the **Three-dot action menu** and select **Delete**.
   You will see a message: "The Section will be deleted for both open and completed To Dos. The To Dos will not be deleted."
3. Click **Delete**.

> [!NOTE]
> What happens when a section is deleted in the checklist?
> - A Section will be deleted from both Open and Completed State irrespective whether it is deleted from Open or Completed state.
> - When a Section is deleted, respective To Dos are out-dented and displayed in the Checklist. They are not deleted from the system.

## Managing To Dos

You can now create To Dos in My Workspace so that you can track and complete activities to manage the investment successfully. These are typically higher-level activities that may not have specific tasks created for them. You can assign an Owner and set due date for a To Do. You can also block and unblock To Dos.

You can perform the following actions in To Dos:
- Creating a To Do
- Assigning a Resource for a To Do
- Setting a Due Date for a To Do
- Managing Notes in To Dos
- Adding/Editing a Link for a To Do
- Managing Attachments for a To Do
- Blocking a To Do
- Marking a To Do as Completed
- Rearranging To Dos

### Creating a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to add To Dos in it.
3. Click **+ To Do** and enter the name of the To Do.
4. A new To Do will be created.

### Assigning a Resource for a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to assign a resource.
3. Click **Assign Owner**.
4. Select a user from the list.

### Setting a Due Date for a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to set a due date.
3. Click **Set Due Date**.
4. Select a date from the Calendar.

> [!NOTE]
> Clarity shows the Due Date with a red background if you select the overdue date.

### Managing Notes in To Dos

You can add notes to To Dos created within Clarity.

#### Adding a Note to a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to add a note.
3. Click **Three-dot Action Menu** and select **Add Note**.
4. Type the text and press **Enter**.

> [!NOTE]
> Some of the key points to remember:
> - You can enter a maximum of 1333 characters in the Note field.
> - Use Shift+Enter to add a line break.
> - To Dos are accompanied by up to three lines of notes. Whenever a note exceeds three lines, an ellipsis will appear.
> - The Note attribute supports field level security. To learn more, see Secure Field Level Access.

#### Editing the Note in a To Do

**Method 01:**

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to edit the note.
3. Click **Three-dot Action Menu** and select **Edit Note**.
4. Edit the text and press **Enter**.

**Method 02:** Click the text in the Note field to edit it.

#### Deleting the Note from a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a note in the To Do.
3. Remove all the text in the Note field to delete the note.

### Adding/Editing a Link for a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to add a link.
3. Click **Three-dot Action Menu** and select **Add Link**.
4. Enter the link in **Edit Link** and click **Done**.

To remove the link for the To Do, follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to remove a link.
3. Click **Three-dot Action Menu** and select **Edit Link**.
4. In **Edit Link**, click **Remove**.

### Managing Attachments for a To Do

#### Upload an Attachment

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In Checklists/Smartlists, click **Three-dot Action Menu** of a To Do and select **Attachments**.
3. Click the **Choose Files** button.
4. Select the file(s) and click **Open**.
   You will view the uploaded file(s) in the Attachments pop-up window.
5. Close the pop-up window. An Attachment icon appears for the To Do.

> [!IMPORTANT]
> You can only upload ten files for a To Do. If you try to upload more than ten files, an error message appears: "Uploading more than 10 files is not allowed".

#### View an Attachment

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In Checklists/Smartlists, click **Three-dot Action Menu** of a To Do and select **Attachments** (or click the Attachment icon of the To Do).
3. Click on the uploaded file name to download and view it.

#### Delete an Attachment

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In Checklists/Smartlists, click **Three-dot Action Menu** of a To Do and select **Attachments** (or click the Attachment icon of the To Do).
3. Click the **Delete** icon in Attachments to remove the uploaded file from the list.

### Blocking a To Do

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, click a To Do to block.
3. Click **Three-dot Action Menu** and select **Block**.
4. Enter the blocked description.

**Ways to unblock To Dos:**
- **Method 01**: Click the **Unlock** button on the blocked To Do.
- **Method 02**: Click **Three-dot Action Menu** and select **Unblock**.

### Marking a To Do as Completed

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In the Checklist, check **Mark Complete** for a To Do.

The To Do moves under the **Completed** list.

### Rearranging To Dos

You can drag and drop To Dos to rearrange them in the Checklist.

## Working with Collaborators

The Checklist can be shared with one or multiple users. The Checklist can only be shared with users defined in Clarity. Collaborator names can be searched by name or email address in the Share Checklists pop-up window. Users with whom the Checklist is shared are added as "Collaborators". Owner and collaborator avatars are displayed next to the Share button, along with the total counter.

> [!NOTE]
> Only owners can share a checklist with others or remove a collaborator.

You can perform the following actions in Checklist:
- Sharing a Checklist
- Modifying Checklist Sharing Group
- Removing a Collaborator
- Roles in Checklists

### Sharing a Checklist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist that you want to share with other Clarity users.
3. Click the **Share** button. The Share Checklist pop-up window opens.
4. Search by name or e-mail address.
5. Select the name/e-mail address to add the user as a collaborator. You can add multiple users as collaborators.
6. Click **Done**.

> [!NOTE]
> Some of the key points to remember are:
> - If you assign an owner and do not share the checklist with the assigned owner then the owner can view the To Do in his/her My Workspace but cannot perform any actions on it.
> - Only the Checklist owner and collaborators can see their checklist. Other users will not see any checklist unless a checklist was shared with them.

### Modifying Checklist Sharing Group

The Checklist Sharing group is seeded in Clarity. Users with whom a checklist is shared are automatically added to this group. This group helps the Administrator to:

1. Add and remove access rights for the group
2. Add and remove users from the group

Follow these steps:

1. Click **Administration**, **Organization and Access**, **Groups**.
2. Click the **Checklist Sharing** group.
3. Open the **Access Rights** menu and click **Global**. You can see the following access rights in Checklist Sharing group:
- Checklist - Create
- My Workspace - Navigate
- Resource - View - All
- To Do - Create - All
- To Do - Delete - All
- To Do - Edit - All
- To Do - View - All

**To add additional access rights:**

1. Click **Add**.
2. Select the appropriate global rights for the user group and click **Add**.
3. If more access rights are available on additional pages, click **Add and Select More**. Continue assigning access rights until all appropriate access rights have been assigned.
4. When you have finished assigning global rights to the user group, click **Return**.

**To remove access rights:**

1. In Checklist Sharing - Global Access Rights, select the access right that you want to remove from the list.
2. Click **Remove**.

### Removing a Collaborator

Follow these steps:

1. Open **My Workspace** in Clarity.
2. Click the Checklist from which you want to remove a collaborator.
3. Click the **Share** button. The Share Checklist pop-up window opens, and you can see the list of Collaborators with an Owner.
4. Click **Three-dot Action Menu** of the collaborator and select **Remove**. The collaborator will be removed from Share Checklist.

### Roles in Checklists

Roles in Checklists define who will take responsibility for performing checklists actions.

| Roles | View | Share | Copy | Rename |
|-------|------|-------|------|--------|
| Owner | Yes | Yes | Yes | Yes |
| Co-Owner | Yes | Yes | Yes | Yes |
| Collaborator | Yes | Yes | Yes | Yes |

> [!IMPORTANT]
> Some of the key points to remember while working as a Collaborator:
> - You can see a collaborator icon for the checklist shared with you.
> - You can view the other collaborators working on the checklist by clicking the **Share** button. It is read-only. You cannot add or remove collaborators.
> - You will become the owner of the copied checklist. You can perform all the checklists actions (View, Share, Copy, Delete, and Rename) on the copied checklist.

## Managing Smartlists

Smartlists help to quickly see the "To Dos" based on conditions. In My To Dos, you can see all To Dos associated with Investments, Tasks, Risks, Issues, Changes and Checklists where you are assigned as Owner and To Dos created by you.

The Smartlists are available in the following layouts:
- List
- Board
- Calendar

Here are some key actions you can perform across all layouts:
- Creating a Smartlist
- Editing the Smartlist
- Deleting the Smartlist
- Sorting To Dos
- Viewing Icons Next to To Dos

### Creating a Smartlist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In **Smartlists**, click **+** to add a new Smartlist.
3. Enter the name of the smartlist.
4. In the **Show To Dos where** section, create the relevant filter for your smartlist.
5. Click **Create**. You see the list of To Dos displayed based on condition.

### Editing the Smartlist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In any layout, click the Smartlist that you want to edit.
3. Click **Three-dot Action Menu** and select **Edit**.

You can rename the smartlist or change the filter condition defined for the smartlist.

### Deleting the Smartlist

Follow these steps:

1. Open **My Workspace** in Clarity.
2. In any layout, click the Smartlist that you want to delete.
3. Click **Three-dot Action Menu** and select **Delete**. The Smartlist will be deleted from the list.

### Sorting To Dos

Across all three layouts, there are three sorting options available in Smartlists: **Date**, **Name**, and **Owner**.
- **Date**: Displays To Dos based on the due date. Categorized under:
- Overdue
- Next 7 Days
- Later
- **Name**: Displays To Dos based on the To Do name.
- **Owner**: Displays To Dos grouped under the user name sorted by their last name.

> [!NOTE]
> Some of the key points to remember while working with Smartlists:
> - You can perform all the To Do actions except creating a new To Do in Smartlist.

## Action Items in My Workspace

In My Workspace, you can use the **Action Items** tab to view all Action Items assigned to you except manual Action Items created in Classic. Both Process and Library Action Items are displayed in this tab. The Action Items are grouped by **Open**, **Pending**, and **Closed**. They are sorted by the most recently initiated Action Item first.

You can take actions directly from this tab using the available action buttons, add comments, and view the decision history for each Action Item. There are no additional access rights required to view this tab. All users who have access to My Workspace can see the Action Items assigned to them.

### View and Respond to Action Items

Action Items assigned to you are displayed in the Action Items tab except manual Action Items created in Classic PPM. By default, the View History panel is collapsed. When expanded, it shows the stage, decision, and lifecycle history of the Action Item along with any user comments. The Date and Time format will be set in **Settings > General** and displayed in the User's time zone (not the server).

Follow these steps:

1. Open **My Workspace**.
2. Click the **Action Items** tab.
- In the right panel, you can see the list of Action items. Each Action Item displays the Subject, Description, Object Type, Object Instance Name, Object Instance Link (for Master Objects only), Priority, and the Action Item action buttons.
- In the left panel, you can see the **Open**, **Pending**, and **Closed** indicating the states of Action Items. You can see a counter to track the number of Action Items belonging to each state respectively:
- **Open**: Action Items awaiting your response.
- **Pending**: Action Items that you have acted on but are waiting for other recipients to respond.
- **Closed**: Action Items that are completed.
3. For each Action Item, you can perform the following actions:
- Click an action button such as **Approve**, **Reject**, or **Acknowledge** to respond.
- If the selected decision requires a comment, the **Add Comment** dialog appears automatically. Enter your comment and click **Save** to complete the action.
- If the decision does not require a comment, you can still add an optional comment by clicking **Add Comment**.
- You can edit or delete your own comment while the Action Item is in Open or Pending status.
- Once the Action Item is closed, comments can no longer be modified or deleted.
- Expand **View History** to see the full decision and comment history for that Action Item. Comments include the author name and timestamp.

> [!NOTE]
> Some key points to remember are:
> - The **Add Comment** dialog appears automatically when a decision is configured to Require comment. You must enter a comment before the system allows you to save the decision.
> - Each user can add only one comment per Action Item.
> - Required comments cannot be deleted.
> - Optional comments can be added, edited, or deleted only when the Action Item is in Open or Pending state.
> - Comments added in Clarity are also visible when viewing the same Action Item in Classic PPM, and vice versa.
> - The Require comment configuration is defined by administrators in the process definition.
> - Open items display both action buttons and the Add Comment option, whereas Pending and Closed items display the decision details and comment history.
> - Supported Master Objects: Projects, Ideas, CITs, and Custom Master Objects. Process Action Items display for supported Sub-Objects.
> - Grouped by Open, Pending, and Closed, sorted by the most recent initiated Action Item first.
> - Assignee can take action by clicking on an action button.
> - No new security rights to view tab or Action Items.
> - The Object Instance Link is available only for master objects (such as Projects, Ideas, or Custom Investments), not Sub-Objects.
> - By default, View History is collapsed. Expand it to review all previous actions and associated comments.
> - Date/Time format will be set in User Settings under General and display in the User's time zone (not the server).

### Action Item Comments Enhancement

In Clarity 16.4.0, Action Items have been enhanced to allow users to add comments directly within Clarity. A new **Require Comment** configuration has been introduced, enabling administrators to make comments mandatory for specific decision actions, such as Reject. When a comment is required, the **Add Comment** dialog appears automatically, and the **Save** button remains disabled until a comment is entered. This enhancement improves decision tracking, audit transparency, and process compliance.

Before you begin using the feature, ensure that you have the following permissions:
- You need to have **Administration - Audit - Navigate** to access the Audit tile.
- Once within **Clarity > Administration > Audit**, you will also require the **Audit Trail - Access** right to view audit details.

## Conversations in My Workspace

You can view a conversation or post your comments in a reply message with optional attachments and links. Conversations are a collection of replies to an original theme or question. Conversations provide a collaborative forum for all team members to discuss, document, and solve specific issues. For example, use conversations to discuss customer feedback or share observations during development. Conversations also help resource managers and investment coordinators discuss which resources to allocate to an investment.

> [!NOTE]
> You can view and reply to conversations that are mentioned (@YourName or @Staff) till the third-level object (for example: Cost Plan Details) in My Workspace Conversations.

This section contains the following topics:
- Pre-requisites
- Panels in Conversations
- View Conversations in My Workspace
- Mention a User Using the @Sign
- Conversations and Replies
- Attach Files in Conversations
- Conversation Notifications
- Difference between Conversations in My Workspace and Notifications
- Access Rights for Conversation-specific Actions

### Pre-requisites

Ensure that you have the following access right to use this feature:
- **My Workspace - Navigate**: Allows user access to my workspace pages in the Clarity. The user will be able to access the checklists to which the user has access.

### Panels in Conversations

You can see two panels in Conversations:

1. **Left Panel**: You can see the list of Conversations where you are mentioned with @YourName and @Staff (you should be a member of the Staff for an investment). A Conversation card in the left panel of the workspace shows:
- The first line from the original conversation for your reference.
- An indicator shows the type of change in the conversation such as:
- You started a new conversation
- You were mentioned
- New reply
- An avatar of the user that started the conversation
- A timestamp to know when the last message was posted to the conversation.

> [!NOTE]
> Some of the key points to remember are:
> - New replies display at the top of the list in the left panel.
> - Only one card per conversation thread.
> - Conversations you reply to that are neither started by you nor mention your name will not appear in My Workspace's Conversation list.

> [!IMPORTANT]
> Click the **Refresh** icon to view the latest Conversations.

2. **Right Panel**: When you click a conversation card in the left panel, you can view the respective conversation details in the right panel. The header shows the conversation parent name and body shows the conversation thread. In the right panel, you can:
- Reply to a conversation
- Attach file
- Edit/delete a conversation
- Navigate to a parent conversation, if you have the required access rights

### View Conversations in My Workspace

You can view all conversations where you are mentioned with @YourName. If you are part of @Staff, you can participate in the conversation. The conversations module includes:
- You started a new conversation
- You mentioned in a conversation
- You mentioned in new reply in a conversation

Follow these steps:

1. Open **My Workspace**.
2. Click the **Conversations** tab.
3. Click a Conversation in the Left panel. You can see the details in the Right panel.
4. Click **Go to <Conversation Parent>** to navigate to the parent conversation page.

> [!NOTE]
> A link on the parent conversation page will be available to navigate back to the My Workspace Conversations.

### Mention a User Using the @Sign

In conversations, use the @ sign to mention other users. Mentioning someone by name sends a new notification to alert them to reply. For example, Derrick and Barb do not mention each other when collaborating to solve staffing issues. They are already communicating in the conversation. However, mention someone in a new conversation or reply to notify them that you would like them to respond.

**Tips for Using the @Sign:**
- You do not have to remember the complete name of the person that you want to mention. After the @ sign, enter the letters of the first or last name. Suggested team members appear.

> [!IMPORTANT]
> You must have the **Resource - View - All** access right to get the list of Clarity users.
- The full name, resource ID, and avatar image appear to verify the identity of the team member.
- All active and locked team members can be mentioned. Inactive team members do not appear and cannot be mentioned.
- We recommend you mention no more than 20 users in a single conversation.

> [!NOTE]
> To mention all users staffed to the investment, including the investment manager, use @Staff. @Staff is not available for Agreements, Roadmaps, Hierarchies, and Custom Objects.

### Conversations and Replies

All conversations display the following behavior:
- The most recent conversation appears first in the right panel.
- If a conversation has more than two replies, click **SHOW ALL** to expand and view the list of remaining replies. Click **SHOW LESS** to view the two most recent replies.
- Only the user who posted a conversation can edit or delete it.

> [!NOTE]
> When you delete a conversation, the application also deletes all replies. You cannot recover a deleted conversation.
- A conversation supports up to 200 replies.
- You can enter a maximum of 1300 characters in a post or reply. Spaces, special characters, line breaks, and @mention strings count toward the 1300-character limit.
- To add a new line in a comment, press **Shift + Enter**.
- Conversations do not support HTML content; however, you can add links to other content. You can copy and paste links or can enter them directly. After you click the **Post** button, the URL appears as a hyperlink. Users can click a link to view the content in a new browser tab. Conversations support the following URL formats: `www`, `http://`, `https://`, `ftp://`, `sftp://`

> [!TIP]
> Do not enter values inside angle brackets in data fields in Classic PPM or Clarity. For example, avoid typing `<abc>` or `<value1>` using angle brackets for placeholder text. The coupled angle brackets are not a supported special character combination and can produce unexpected results.

### Attach Files in Conversations

You can upload and edit up to five attachments to each reply in a conversation. Team members can download and view the attached files.

Follow these steps:

1. Open a conversation. A **Reply...** prompt appears for the next comment.
2. In the new reply, click **Attach**. To add attachments to a previous comment, click the **Options** menu and select **Edit** and then click **Attach**.
3. Select the file you want to attach. Press and hold down the **Ctrl** key to select up to five files. Click **Open**. The thumbnails of the attached files appear below the comment.
4. Add or edit your reply text. You cannot attach a file without entering an associated comment.
5. Click **Post** or press the **Enter** key. The file attachments are uploaded.
6. (Optional) To delete an attachment, click the **Options** menu and select **Edit**. Click **X Delete**.

> [!NOTE]
> - When you delete a reply, the application also deletes any attachments.
> - When you delete an investment, a task, a risk, an issue, a change request, an agreement, a roadmap, a hierarchy, or a custom object, the application deletes any associated conversations, including attachments.

> [!TIP]
> File attachments follow the same restrictions as project documents. As an administrator, you determine project document settings. For example, the maximum size of an attachment is 20 MB. To limit acceptable file types, you can also set the **Authorized File Extensions** field. See Configure General System Settings.

### Conversation Notifications

When a user mentions you or your team, notifications alert you about conversation activity.
- To view your notifications, click the notifications bell icon at the top of all pages in Clarity.
- The number of new notifications appears on the notifications bell. The number disappears when you click the bell.
- The notifications panel displays the avatar of the sender and the first line of their message.
- Scroll up or down to view all notifications on the panel.
- To open a notification, click one in the list. Supporting details about the context of the conversation appear. You can reply, edit, or delete the message. You can also navigate to the specific page from the notification message window. Click **Go to <Conversation Parent>**.
- If a user is mentioned more than once in a single message, they are notified only once.
- If a team is mentioned (@Staff) in an investment conversation, a notification is sent to all the team members, including the investment manager.
- To delete a specific notification, hover over the notification and click **Delete**. To delete all notifications, click **CLEAR ALL**. The notifications are deleted for the specific user.
- If a conversation is deleted, the notification shows a message: "This message has been deleted".

> [!TIP]
> - You can personalize the way that you receive notifications. By default, the application sends you conversation notifications by e-mail, SMS text message, and on-screen. See Set Up Notifications in Personalize Clarity: Change Password, Account Settings, Notifications, Export to Excel.
> - As an administrator, you can modify the two notification templates for conversations. These templates define how notifications appear in the application:
>   - Conversation - Mentioned in a conversation
>   - Conversation - Mentioned in reply to a conversation
>   See Configure Notifications and Notification Templates.
