As a team member, you are responsible for tracking the number of hours that you work on specific tasks. Update your task status. Use timesheets to track your work hours. Accurate time reporting provides your company with valuable data for project cost tracking and charge code analysis.

Team members can also view all tasks assigned to the team. If you login as an individual user who has access to timesheets, you can see the tasks assigned to you and the team tasks for which you are a member. Individual team members can now enter time for tasks assigned to their team. The resource manager, project manager, or administrator will be able to see that the individual has updated the timesheet.

## Update Task Status

As a team member, you communicate status updates, risks, and issues to your team and to the project manager. Balance multiple tasks with different specifications and deadlines, deliver quality work, and complete tasks on time.

> [!NOTE]
> Update your task status several times daily and always before submitting your timesheet.

Follow these steps:

1. Open an investment and click **Tasks**. All the tasks are displayed.
2. Search for a task or select a task by using the filter option.
3. Select a task to open the Details pane.
4. Update task details. The following options are typically updated:
   a. **To Do:** Add or remove action items in a To Do list. You can also mark to do items as completed.
   b. **Conversations:** Start a conversation or post a reply about a task.

## Create a Timesheet

You can add work items by selecting the tasks to appear on your current timesheet. You can also duplicate a previous timesheet to record new hours against the same entries.

> [!NOTE]
> Not all timesheet creation options are always available. For example, the option to create from a previous timesheet is unavailable if you have not created one yet. The option based on current work assignments is unavailable if you have no current assignments.

### Create a Timesheet with Previous Timesheet Tasks or Current Assignments

Follow these steps:

1. Click **MY TIMESHEETS**.
2. Open a timesheet period.
3. Click **Add Tasks**.
4. Select an option from the list:
- **Add Assigned Tasks:** Enabled only when you have assignments. Populates a timesheet with task assignments currently open. The open assignments have a date range within the time period of the new timesheet and typically have non-zero pending or accepted ETC. The **Populate Time Range** field includes a setting for the days before and after the time period start (default = 7 days). If any open assignments have non-zero ETC, tasks in future time periods outside the range also appear.
- **Add Assigned Tasks (with ETC):** Automatically populates timesheets based on assignments using the Estimated Time to Complete (ETC).

> [!NOTE]
> The **Add Assigned Tasks (with ETC)** option is enabled only if:
> - Your administrator enables the **Populate Assigned Tasks (with ETC)** option in Timesheet Options.
> - For Investments: Ensure the resource is assigned and ETC is set for the tasks.
> - For Team Investments: Assign resources and specify the default allocation percentage.
- **Copy Previous Timesheet:** Creates a timesheet with the task entries from your most recently submitted timesheet.

> [!NOTE]
> Enabled only if your administrator has enabled the **Copy time entries from previous timesheet** option and you have created a previous timesheet. If the previous timesheet is more than four periods in the past, this option is unavailable.
- **Copy Previous Timesheet (with time):** Includes the actual hours from the previous timesheet with two exceptions: it does not copy actuals for incidents, and it does not copy time entries for one-time actuals (such as vacation or sick time from non-working days).

> [!NOTE]
> Enabled only if your administrator has enabled the **Include actuals from previous timesheet (except incidents)** option and you have created a previous timesheet.
- **Select Tasks:** Provides a list of work items available for selection.

> [!IMPORTANT]
> - After creating a timesheet, you can add more tasks using the **Add Tasks** button.
> - You will have a link on **Task Name** in My Timesheet to navigate back to Task Properties.

> [!TIP]
> As an administrator, you define the date range that determines which tasks appear on user timesheets. You can configure the time entry options for all timesheets.

### Create Timesheet by Executing the Populate Timesheets Job

The Populate Timesheets job automatically fills timesheets based on assignments.

1. Click **Home**, **Personal**, **Reports and Jobs**.
2. Click **Jobs**, **Available Jobs**.
3. Click the **Populate Timesheets** job.
4. Enter the required fields.
5. Click **Submit**.

**Prerequisite:** Ensure the **Allow submission of timesheets with no tasks** option is enabled in Timesheet Options.

1. Click **MY TIMESHEETS**.
2. Open a timesheet period.
3. Click **Submit**.

## Impact on My Timesheet Options due to Assignment's "Open for Time Entry" (OTE) Field

| Task | Assignment OTE | Add Assigned Tasks | Select Tasks |
|---|---|---|---|
| Task 01 | Enabled | Task will be added to the Timesheet | User can view and add Task to the Timesheet |
| Task 02 | Disabled | Task will not be added to the Timesheet | User can only view Task on the Timesheet |

> [!NOTE]
> If the Assignment "Open for Time Entry" field is enabled, but the Project/Task/Team "Open for Time Entry" field is disabled, then the user cannot add the Task to their Timesheet.

### Timesheet Options and Assignment's OTE

| Timesheet Option | Assignment OTE Disabled Impact |
|---|---|
| **Populate assigned tasks** | Clarity does not add the Task. If disabled for all assigned tasks, the "Add Assigned Tasks" option is disabled. |
| **Copy time entries from previous Timesheet** | Clarity does not copy tasks with disabled OTE. If disabled for all tasks, an error message is displayed. |
| **Include actuals from previous Timesheet (except incidents)** | Same behavior as above. |

> [!IMPORTANT]
> If the Assignment "Open for Time Entry" field is enabled, the Timesheet Options will function as expected.

## Identify Manual and Automated Time Entries

You can identify whether time entries were created manually or automatically. Use the **Entry Method** and **Job Type Code** fields to better understand and audit how time entries are populated and modified.

### Entry Method
- **Null/Empty:** Entry was created before the Clarity 16.3.3 upgrade.
- **Manual:** Entry was created or modified by a user (selecting tasks, copying assignments, editing charge codes, lookup fields, or actuals).
- **Automated:** Entry was created by a system job such as Timesheet Integration or Populate Timesheets.

> [!NOTE]
> If a user modifies an automated entry, the Entry Method is updated to Manual. However, the original Job Type Code value is retained.

### Job Type Code

Displays the name of the job that created or updated the entry (e.g., `timesheet_integration`, `populate_timesheet_using_etc`).

> [!NOTE]
> The Job Type Code displays null for manual entries or entries created before Clarity 16.3.3.

**Prerequisites:** Ensure **Entry Method** and **Job Type Code** are included in the Selected Columns under Timesheet Options.

**Steps to Configure Columns:**

1. In the Clarity Main Menu, navigate to **Timesheets**.
2. Click **Columns**.
3. In the Columns panel, select **Entry Method** and **Job Type Code**.

### Scenarios

| Scenario | Entry Method | Job Type Code |
|---|---|---|
| Created by Populate Timesheet job | Automated | Populated |
| Created manually or copied | Manual | null |
| Automated entry modified by user | Manual | Retained |
| Created before Clarity 16.3.3 | null | null |

> [!IMPORTANT]
> **Timesheet Rules Support:** Use Entry Method and Job Type Code attributes to configure validation rules (e.g., prevent manual changes to specific tasks, allow only automated entries for specific charge codes).
> **Field-Level Security (FLS):** Secure or relabel the new attributes using **Administration > Attributes**.
> **Reporting and Data Warehouse Support:** These attributes are available in Data Warehouse, Data Provider, and Report Design.

## Create a Timesheet by Adding Work Items from Active Investments

As a new team member on a project with no assignments for the current period, create a timesheet by selecting individual work items.

1. Open the timesheet period.
2. Select **Add Tasks** and click **Select Tasks** to open the Select Tasks generic grid.
3. Filter the list of tasks. By default, the list shows all tasks on investments where you are the team member.
4. Select the checkboxes next to the tasks you want to add. Click **Add**.
5. Enter the number of hours you worked on each task each day.
6. Click **Add Tasks** to add additional tasks. Select an option: Select Tasks, Add Assigned Tasks, Copy Previous Timesheet, or Copy Previous Timesheet (with time).

> [!NOTE]
> When you add additional tasks using Add Assigned Tasks/Copy Previous Timesheet/Copy Previous Timesheet (with time), a pop-up appears: "Tasks with no time entered will be removed. Do you want to continue?"

7. Click **Remove Tasks** to remove multiple task line items.
   a. Select the checkboxes or click **Select All**.
   b. Click **Delete And Close**.

**Other Options:**
- Click the **more options** icon next to a task line item to **Copy** or **Delete** it.
- Click **Expand All/Collapse All** to expand and collapse all tasks.

> [!IMPORTANT]
> You can enter work time only if:
> - Investments and tasks are Open for Time Entry
> - You are an active user
> - You are part of the labor resource class
> - A default filter (Assigned Resource = Current User) is applied when using Select Tasks

> [!NOTE]
> Clarity does not include the following tasks in **Add Tasks**:
> - Milestone Tasks
> - Phases (Summary Tasks)
> - Tasks of inactive investments
> - Tasks where Track Mode is not set to PPM
> - Tasks where Open for Time Entry is set to false
> - Tasks for investments set as templates
> - Tasks for programs

## Enter Hours on a Timesheet

Record the hours that you worked each day on specific task assignments. When you have completed all entries, click **Submit**. Your submitted timesheet goes to your manager for approval.

The timesheet allows you to enter more than 24 hours per day. This flexible design allows you to enter an aggregate amount on one date spanning multiple days.

> [!NOTE]
> If your administrator configured time entry display unit as Days in Classic PPM, your timesheets show and let you record time in days.

### Timesheets and Non-Working Days

Non-working days are indicated with a gray column. You can enter an amount in the column for non-working days.

### Non-Project Work Time (Holidays, Personal Leave, Vacation Time)

Your project manager or resource manager can set up an **Other Work** investment type in Classic PPM to track non-project hours. You can select it as a work item and record holidays, personal leave, and vacation time.

> [!TIP]
> If using Other Work to track personal time or vacation, mark those days as non-working days on your calendar to prevent the hours from being copied to your timesheet.

### Timesheets and Estimate-to-Complete (ETC)

Your administrator can include an ETC column on your timesheet. When you override this value, it represents a pending ETC. The color for the number changes to bold and blue, and hovering shows the original ETC value.

**To Accept or Reject Pending ETCs in Clarity:**

1. Open an investment.
2. Click the **Assignments** module or **Tasks** module (then click **Details**, **Assignments**).
3. Click **Columns** and check the **ETC** and **Pending ETC** fields.
4. To **accept**: Copy the Pending ETC amount into the ETC cell, then delete the Pending ETC value.
5. To **reject**: Delete the value in the Pending ETC cell.

## Enter Time for Others

As a resource manager or team member with approval rights, you can create, edit, and submit timesheets for other team members when these criteria are met:

2. Select the required time period.
3. Click the **Select resource to enter time** icon. A list of team members appears.
4. Click **View** to select the resource.
5. Click **Add Tasks** and choose relevant options.
6. Enter the hours for the team member.
7. Click **SUBMIT**.

## Frictionless Time: Create Timesheets without Staff or Assignments on Investments

When you enable the **Allow Time Entry for Unstaffed Resources** field in the Investment workspace, you can create and post a timesheet without assigning Staff or Assignments.

**Impact:**
- An effort task is automatically created if no existing tasks are present.
- You do not need to be allocated to the Staff module, nor do you need any assignments.

**Prerequisite:** Ensure view access to the Investment instance.

Follow these steps:

1. Open the **Projects/Investments** workspace.
2. Configure the **Allow Time Entry for Unstaffed Resources** field from the Column Panel.
3. Enable the field for a specific Project or Custom Investment (disabled by default).
4. In Clarity, click **Timesheets**.
5. In My Timesheets, click **Add Tasks**, then **Select Tasks**.
6. Apply filters: Remove Assigned Resources = Current User; Add Investment Name = specific investment.
7. Select tasks and click **Add**.

> [!NOTE]
> This is an effort task with Task ID `~rmw`. You can complete the entire timesheet lifecycle using this task.

> [!IMPORTANT]
> The Allow Time Entry for Unstaffed Resources setting does not override existing rules. The Open for Time Entry and Track Mode settings still apply.

## Frictionless Time: Create Timesheets from Rally for a Specific Time Period

### Prerequisites
- Created the Rally integration instance in Classic PPM.
- Defined the Financial Work Types lookup values in Classic PPM.
- Created and activated a Frictionless Accounting Mappings instance in Clarity Administrator.
- Created an Integration Mappings instance with Object and Field Mappings.
- Enabled the **Allow Time Entry for Unstaffed Resources** field.

### Steps

1. Open the Project/Custom Investment workspace.
2. Configure the following columns:

| Field | Value |
|---|---|
| Integration Mapping | Set to the Integration Mapping instance value |
| Rally Object | N/A (populated after sync) |
| Rally Formatted ID | N/A (populated after sync) |
| Allow Time Entry for Unstaffed Resources | Enabled |
| Open for Time Entry | Enabled |
| Track Mode | PPM |

> [!NOTE]
> - Honor the "Open for Time Entry" setting on Frictionless Work Type tasks.
> - Rally Object and Rally Formatted ID are populated after running the Synchronize Mapped Information job.

3. In Classic PPM, navigate to **HOME, Reports and Jobs, Jobs**.
4. Execute the **Synchronize Mapped Information** job.
5. Log in to Rally using the Rally Integration settings.
6. Create user stories under the feature with Financial Work Types.
7. In Classic PPM, execute the **Timesheet Integration** job with Integration = FCA and the current time period.
8. Verify that tasks with Financial Work Types appear correctly in the user's Clarity Timesheet.

> [!NOTE]
> After the Timesheet Integration Job runs, the Financial Work Type's Task ID displays the Financial Work Type Lookup ID values.

9. Submit the timesheet for approval.
10. After approval, execute the **Post Timesheets** job in Classic PPM.

After posting, the following changes occur:
- Financial Work Types are converted into tasks within the investment.
- The user is added as Staff.
- Assignments display the user details.
- Actuals and Actual Cost are populated for frictionless timesheets.
- Task IDs are updated in the Timesheet.

## How Rally Calculates Hours for Financial Work Types

When the Timesheet Integration job runs, Rally converts story points into timesheet hours in three steps:

1. **Find & sum work items:** Rally totals the Plan Estimate of every user story that is In Progress or Accepted, carries a valid Financial Work Type, and is not deleted.
2. **Derive work ratios:** Points are grouped by Feature and Financial Work Type; each group's total is divided by the project's overall total to produce a percentage (work ratio).
3. **Allocate team-member hours:** Rally multiplies every work ratio by each team member's available hours for the period.

## Frictionless Time: Auto-submit Timesheets

The Auto-submit Timesheets feature automates the submission of timesheets in Open or Returned status.

**Workflow:**

1. Execute the Auto-submit Timesheets job.
2. The job validates timesheet rules configured in Clarity.
3. If errors exist: the resource receives a notification, the **Errors** button appears in My Timesheet, and the **Has Errors** field is checked.
4. If no errors: the timesheet is automatically submitted.

Follow these steps:

1. In Classic PPM, navigate to **Home > Reports and Jobs > Jobs**.
2. Click the **Auto-submit Timesheets** job.
3. Enter the required fields and click **Submit**.

## Review and Approve Timesheets

As a resource manager, you review and approve timesheets for resources that report to you. Timesheet review and approval helps:
- Monitor budget and resource allocation
- Manage time on the schedule
- Measure team member performance
- Measure planned project against actuals

### Send Timesheet Reminders

As a timesheet approver, send reminders from the **Open** and **Returned** columns.
- A reminder sent icon appears with the time that the reminder was sent.
- A reminder notification is sent based on Classic PPM configuration.
- After sending a reminder, a **Remind Again** button is available.

### Add Notes to Timesheets

Add generic notes at the timesheet level (e.g., reminders to submit) by clicking the team member name, then **Notes** in the Details panel.

### Add Notes to Timesheet Entries

From the detailed view, add a specific note for an individual task line item.

### Review, Approve and Return Submitted Timesheets

**From Review & Approve Tab:**

1. Click **Timesheets** in the main menu.
2. Go to **Review & Approve** tab.
3. Click the resource name.
4. To approve: click the thumbs-up icon.
5. To return: click the thumbs-down icon.

**From Timesheets Tab (Bulk Actions):**

1. Click **Timesheets**, go to **Timesheets** tab.
2. Select **Time Period is Open** from the Column Picker.
3. Apply filters (e.g., Status = Submitted).
4. Select timesheets and click **Approve** or **Return**.

> [!NOTE]
> Bulk actions appear only if you select timesheets with Submitted or Approved status.

### Adjust Posted Timesheets

Occasionally a posted timesheet needs adjustment. Click **Adjust** to create an adjustment timesheet, which appears as a new Open timesheet.

> [!NOTE]
> The drop-down on the Adjustment Timesheets bar displays all dates on which the timesheet was adjusted.

### Search or Filter Timesheets

Use the Search or Filter bar to find timesheets by:
- **Resource:** All resources for whom you have approval rights
- **Resource Manager:** Resource managers for your approved resources
- **Resource OBS:** All resources associated to an OBS

### Access Timesheets from Rally

As a team member, you can access Clarity timesheets directly from Rally. Contact your administrator for configuration.

> [!NOTE]
> The timesheet integration feature is not supported if you are using Clarity SaaS with the On Demand Portal.

## View and Manage Timesheets From Grid Layout

The Timesheets grid provides Grid-Filter-View (GFV) capabilities including group, sort, filter, save views, and export to CSV.

### Prerequisites
- At least one task or work item must be associated with the resource.
- **Open for Time Entry** is checked and **Track Mode** is set to PPM.
- Required access rights: **Timesheets - Navigate**, **Resource Enter Time**, **Resource Approve Time**, **Timesheet - Approve All** (global), or **Timesheet - Edit All** (global).
- Configure necessary modules from the **Timesheet Blueprint**.

### Show or Hide the Column Panel for MY TIMESHEET

1. In Classic PPM, navigate to **Administration, Project Management, Timesheet Options**.
2. Select or clear the **Allow End user column configuration** checkbox.
3. Click **Apply to All Resources**.

### Specify Columns from Column Panel

When the checkbox is selected, configure columns in Clarity by clicking the **COLUMNS** button on any user timesheet.

### Specify Columns from Classic PPM

When the checkbox is cleared, configure columns from the **Default Content and Layout** section in Classic PPM Timesheet Options.

### Display Effort Gauge and Project Time Gauge

In Classic PPM Timesheet Options, under **Modern UX Options**, select **Display Effort Gauge** and **Display Project Time Gauge**.

### View Timesheets in Grid Layout

1. Click **Timesheets**, then the **Timesheets** tab.
2. View timesheets from the grid with filtering, sorting, grouping, and saved views.
3. Select a row to open the Details panel with Timesheet, Notes, and Details tabs.
4. Click **GO TO TIMESHEET** to view in MY TIMESHEET tab.
5. Click **REVIEW** to view the resource timesheet detail page.

> [!IMPORTANT]
> If a user has submitted the timesheet, a link to the Resource Timesheet - Detail modal appears in the grid (only for Submitted status).

### View All Timesheets of All Statuses

You can view all open timesheets created based on the time period assigned for tasks. Required access rights: Resource Enter Time, Resource Approve Time, Timesheet - Approve All, Timesheet - Edit All.

### Resource OBS Filter for Timesheet

The Resource OBS filter allows filtering timesheets by resource OBS to locate resources from specific locations.

> [!NOTE]
> The resource OBS is only available in the filters.

### Employment Type Lookup Attribute in Timesheet Grid

The Employment Type attribute lets you filter resources based on whether they are employees or contractors. You can view it in the grid and Details panel.

### View Timesheets for Closed Time Periods

1. Navigate to **Timesheets**, click **TIMESHEETS**.
2. Click **Filter**, **Add Filter**, and set **Time Period Is Open = No**.
3. Select a row to view the closed timesheet (read-only).

> [!TIP]
> Timesheets that have never been opened or edited will not appear in the Clarity Timesheet Grid (these are virtual timesheets). To view them, you must open the respective period for time entry.

### View and Filter Error Timesheets using 'Has Errors'

1. Click **Timesheets** from the Main Menu.
2. Navigate to the **Timesheets** tab.
3. Click the **Columns** panel and select **Has Errors**.

> [!NOTE]
> If there is an error in an Open or Returned timesheet, the Has Errors field is checked.
