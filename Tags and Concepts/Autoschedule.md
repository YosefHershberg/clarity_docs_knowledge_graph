# Autoschedule

Autoschedule automatically calculates task dates based on dependencies, constraints, and resource calendars.

## How It Works
1. Define task dependencies (Finish-to-Start, Start-to-Start, etc.)
2. Set task constraints (Must Start On, As Soon As Possible, etc.)
3. Define task effort estimates (ETC)
4. Run Autoschedule to calculate optimal dates

## Key Concepts
- **Critical Path** — The longest path through the schedule determining minimum project duration
- **Float/Slack** — Time a task can be delayed without affecting the project end date
- **Resource Calendars** — Working days and hours affect schedule calculations
