# Personalize Dashboards and Portlets

## Favorites Menu

### Add a Favorites Menu Section

You can add a section to the **Favorites** menu to organize your saved favorite pages.

**Steps:**
1. Open **Favorites** and click **Configure**.
2. Click **Add**.
3. Select **Section** and click **Next**.
4. Complete the requested information.
5. Save your changes.

### Reorder Menu Sections

You can reorder sections and links in the **Favorites** menu.

**Steps:**
1. Open **Favorites** and click **Configure**.
2. Click **Reorder**.
3. Select a section in the **Menu Items** list and click the arrows to move it to a new position.
4. Save your changes.

> [!NOTE]
> If you do not see your changes reflected in the menu, click the **Refresh** button.

### Remove a Menu Section or Link

You can remove a menu section, page link, or action link. You can only remove sections or links that you have created. You cannot remove the Classic PPM default sections or links.

To remove a section or link, open **Favorites**, click **Configure**, and select and remove the item.

## Dashboards

You can use personal dashboards to display specific information. You can create tabbed dashboards with multiple pages, and add stock or custom portlets. You can share dashboards, export them to Microsoft Excel or PowerPoint, and view dashboards shared by other users.

### Dashboard and Portlet Differences

The following table highlights functional differences between dashboards and portlet pages.

| Functionality | Dashboard | Portlet Page |
|---|---|---|
| Exported output | One portlet per slide or sheet | Fit to page |
| Sharing with specific users | Yes | No |
| Page type | Page with tabs, Page without tabs | Page with tabs, Page without tabs |
| Portlet layout | Two-column, three-column templates | Two-column, three-column templates, row layout |

### Dashboard Examples

**Example 1: Project Status**

Karen creates a tabbed dashboard to track multiple projects. Her portlets include:
- Project Risks
- Budget
- Resource Allocation
- Milestones

She configures the layout by dragging and dropping portlets, adds a shared filter, and exports results to PowerPoint for weekly team meetings.

**Example 2: Task Status**

Roberto, without dashboard creation rights, navigates to **Dashboards** and views two shared dashboards: **Security Development Team** and **Weekly Time Entries**. He can view his assigned tasks and click through to work on them.

**Example 3: Team Status**

Stan creates a two-tab dashboard (**Team Work** and **Team Status**) populated with stock portlets: Team Members, Bugs and Issues by Components, Tasks by Team Member, Time Entries by Team member, and Work Completed and Work Remaining.
He shares it with the entire OBS development unit and executive management.

### Dashboard Prerequisites

Your administrator assigns the following global access rights:

- **Dashboard
- Create** ג€” Allows you to create a dashboard.
- **Dashboard
- Navigate** ג€” Provides access to the Dashboards link in the Personal menu.
- **Portlet
- Create** ג€” Allows you to create a portlet.
- **Portlet
- Navigate** ג€” Provides access to the Portlets link in the Personal menu.

> [!NOTE]
> Access to a dashboard does not automatically include access to the project information displayed on it. Users must also have the appropriate access rights to that information.
Sharing a dashboard with a user does not automatically enable them to see it ג€” the user must also have the **Dashboard - Navigate** right.

Before setting up a dashboard, consider:
- The number of portlets you plan to include (helps determine if you need tabbed pages).
- How portlets will present information (graphs vs. statistical rows/columns).
- Which users you can share with and whether they have rights to the displayed information.

### Create a Dashboard

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Click **New**.
3. Complete the requested information:
- **Type** ג€” Specifies if the dashboard is a single page or tabbed pages. Select **Page with Tabs** to add tabs.
- **Personalizable** ג€” Specifies if users you share with can modify their copy. Personal changes are local to each user and are overwritten when the owner publishes new changes.
4. Save your changes.

### Configure the Dashboard Layout

A layout template determines the number of portlets per row and their percentage width. If a portlet exceeds its allotted space it will not be truncated, but the displayed dashboard may differ from the design view.

From the **Dashboard Details** page you can:
- Create portlets.
- Add stock portlets or portlets you created.
- Select a layout template.
- Drag-and-drop portlets into position.
- View, export, or share the dashboard.

#### Configure a Dashboard Without Tabs

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page for the dashboard.
3. Click the **Add Portlet** icon in the **Content and Layout** section.
4. Select the check box for each portlet and click **Add**.
5. Select the layout template (mouseover each option to see its column format).
6. Drag-and-drop to arrange portlets.
7. Save your changes.

#### Configure a Dashboard With Tabs

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page for the dashboard with **Page with Tabs** type.
3. Click the **Add Tab** icon in the **Content and Layout** section.
4. Complete the requested information and save your changes.

#### Add a Portlet to a Dashboard

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page for the dashboard.
3. Select the tab to configure in the **Contents and Layout** section.
4. Click the **Add Portlet** icon.
5. Select the check box for each portlet to add and click **Add**.
6. Select the layout template.
7. Drag-and-drop portlet icons into position.
8. Save your changes. Repeat for each tab.

### Share a Dashboard

You can share a dashboard with a user, a group, or an OBS group. By default, shared users have view-only rights. You can also assign manager rights, which allow the user to change and publish the dashboard.

> [!NOTE]
> If a user personalizes a shared dashboard and the owner (or a manager) later publishes changes, the user's personal changes are overwritten.

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page, open the **Actions** menu, and click **Sharing**.
3. Click the appropriate menu to add a resource, group, or OBS unit. Click **Full View** to see all users who have access.
4. Click **Add**.
5. Select the individuals, groups, and OBS groups to share with and click **Add**.
6. To assign manager rights, select a user and choose **Manager** from the **Access Right** column.
7. Save your changes.

### Personalize a Shared Dashboard

You can personalize a shared dashboard if you have **Manager** access. Your changes are visible only to you and will be overwritten if the dashboard owner publishes new changes.

You can personalize:
- Dashboard name and description.
- Dashboard content (add/remove portlets).
- Page filters for portlets.
- Portlet layout.

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page, open the **Actions** menu, and click **View Dashboard**.
3. Click **Personalize**.
4. Select the appropriate menu and modify the dashboard.
5. Save your changes.

### Publish a Dashboard

A dashboard manager can publish changes so that every user who can view the dashboard sees the new version. This overwrites any personalizations users have made.

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Click the name of the dashboard to change.
3. Make changes if necessary and click **Save**.
4. Click **Publish**, then click **Yes** to confirm.

### Export a Dashboard

You can export a dashboard or individual portlets to Microsoft Excel or PowerPoint. The maximum export size is 300 records (no limit for **Export to Excel (Data Only)**).

## Export options:
- **One Portlet per Slide/Sheet** ג€” Exports each portlet to a separate page at the current dashboard size.
- **One Portlet per Slide/Sheet** + **Fit to Page** ג€” Exports each portlet to a separate page, fitted to the page.
- Neither checked ג€” All portlets export to a single sheet (some may be truncated in PowerPoint).

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page.
3. Select the appropriate **Export** check boxes.
4. Open the **Actions** menu and click **Export to Excel** or **Export to PowerPoint**.

### Export a Portlet

> [!NOTE]
> Some stock portlets are restricted and cannot be exported. Chart portlets export in native Microsoft Office chart format.

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page, open **Actions**, and click **View Dashboard**.
3. Open the **Options** menu on the portlet and select one of:
- **Export to Excel (Data Only)** ג€” Portlet data without charts.
- **Export to Excel** ג€” Portlet data and charts.
- **Export to PowerPoint** ג€” Portlet data and charts.
4. Click **Open** to view without saving, or **Save** to save the file.

### Remove a Portlet from the Dashboard

> [!NOTE]
> Removing a portlet from a dashboard does not delete it. To permanently delete a portlet, remove it from the portlets list page.

**Steps:**
1. Open **Home**, and from **Personal**, click **Dashboards**.
2. Open the details page.
3. Click the **Remove** icon on the portlet in the **Content and Layout** section.
4. Save your changes.

## Personalize a Portlet

You can configure the appearance of data in list, chart, or filter portlets.

### Configure a List or Grid Portlet

#### Set List Column Layout

**Steps:**
1. Click the **Options** icon on the portlet toolbar.
2. Open the **Options** menu and click **Configure**.
3. In the **Column Layout** section, select columns from the **Available Columns** list and click the arrow to move them to the **Selected Columns** list. Use **Move Up** and **Move Down** to reorder.
4. In the **Column Sorting** section:
- Select a field in the **Column** drop-down to set sort order.
- Click **Ascending** or **Descending**.

> [!NOTE]
> Column sorting is not available for hierarchical list pages.

5. Save your changes.

#### Set List Options

**Steps:**
1. Open the **Options** menu on the portlet toolbar and click **Configure**.
2. Open the **List Column Section** menu and click **Options**.
3. Complete the fields:
- **Secondary Value Display** ג€” Controls how secondary values appear in a grid cell:
- *Mouseover only* ג€” No secondary value displays.
- *Mouseover and redline text* ג€” Secondary value displays on cursor hover.
- *Show Null Secondary Values* ג€” Displays the secondary value even when empty.
- **Filter** ג€” Controls how results appear initially.
- **Display Currency Code in Column** ג€” Shows the currency code for money attributes.
- **Allow Configuration** ג€” Users can modify portlet appearance.
- **Allow Label Configuration** ג€” Users can modify portlet labels.
4. Save your changes.

#### Add a Gantt Chart to a List Portlet

A Gantt chart is a virtual field showing duration and progress over time. New virtual fields appear in the far-right column by default.

**Steps:**
1. Open the **Options** menu on the portlet toolbar and click **Configure**.
2. Open the **List Column Section** menu and click **Fields**.
3. Click **New**.
4. Select **Gantt** and click **Next**.
5. Complete the **General** section.
6. Complete the **Time Scale** section:
- **Time Scale** ג€” Time values shown across the top of the chart.
- **Number of Time Periods** ג€” Number of periods to display.
- **Time Period Offset** ג€” Shifts the start of the Gantt bar relative to the Start Date.
- **Show Group Row Header** ג€” Prints the time scale value.
7. Complete the **Primary Bar** section:
- **Item Name Attribute** ג€” Field name for the primary bar.
- **Start Date Attribute** ג€” Date field for the bar's start.
- **Finish Date Attribute** ג€” Date field for the bar's end.
- **Milestone Attribute** ג€” Field for milestone; a non-zero value displays a diamond.
- **Progress Through Date Attribute** ג€” Date to indicate completed work (overrides Progress Percent).
- **Progress Percent Attribute** ג€” Percentage to move the green progress line (overrides Progress Through Date).
- **Label for Bar** ג€” Text to display above each task bar.
- **Additional Information Attribute** ג€” Field shown in the mouseover tooltip.
- **Show Mouseover** ג€” Items displayed when hovering over a bar.
8. Select dates to show on the primary bar from the **Available** list and move to **Selected**.
9. (Optional) Select **Show Secondary Bar** to display a comparison bar below the primary bar.
10. Save your changes.

#### Configure the Gantt Chart Display Settings

**Steps:**
1. Open the **Options** menu and click **Configure** on any page with a Gantt chart.
2. Open the **List Column Section** menu and click **Fields**.
3. Click the **Properties** icon next to the column label (e.g., **Schedule**).
4. Configure the display settings.
5. Save your changes.

#### Modify Time Period Settings for a Gantt Chart

**Steps:**
1. Open the **Options** menu and click **Gantt** on a page with a Gantt chart.
2. Modify the fields:
- **Start Date** ג€” Start date for the column spread.
- **Time Scale** ג€” Period by which data displays (days, weeks, etc.).
- **Number of Time Periods** ג€” Number of periods to display.
- **Time Period Offset** ג€” Shifts the start of the Gantt bar; enter a positive or negative number.
3. Save your changes.

#### View and Edit List Portlet Fields

**Steps:**
1. Open the **Options** menu and click **Gantt** on a page with a Gantt chart.
2. Open the **List Column Section** menu and click **Fields**.
3. Use the **Show** and **Display** fields to control what you see.
4. Click the **Properties** icon to change how a field displays.
5. Enter a new value in the **Column Label** field to change a field label.
6. Save your changes.

#### Add an Image to a List Portlet

**Steps:**
1. Open the **Options** menu on the portlet toolbar and click **Configure**.
2. Open the **List Column Section** menu and click **Fields**.
3. Click **New**.
4. Select **Image** and click **Next**.
5. Complete the **General** section:
- **Image** ג€” Image to use in the grid column.
- **Link** ג€” Page link that appears as a secondary value.
- **Open as Pop-up** ג€” Whether the linked page opens as a pop-up.
6. Save your changes.

#### Add a Progress Bar to a List Portlet

**Steps:**
1. Click the **Options** icon on the portlet toolbar.
2. Open the **Options** menu and click **Configure**.
3. Open the **List Column Section** menu and click **Fields**.
4. Click **New**.
5. Select **Progress Bar** and click **Next**.
6. Complete the fields:
- **Current Stage Name** ג€” Field value for each stage; displays below the column label.
- **Current Stage Number** ג€” Field value for the current stage.
- **Number of Stages** ג€” Field value defining the total stages.
- **Show Label** ג€” Displays the current stage name in the progress bar.
7. Save your changes.

#### Add a Time Scaled Value to a List Portlet

> [!NOTE]
> Only administrators with the appropriate rights can create time-varying attributes. You can display multiple time-scaled values stacked in the same virtual column.

**Steps:**
1. Click the **Options** icon on the portlet toolbar.
2. Open the **Options** menu and click **Configure**.
3. Open the **List Column Section** menu and click **Fields**.
4. Click **New**.
5. Select **Time Scaled Value** and click **Next**.
6. Complete the **General** section:
- **Value Attributes** ג€” Time scaled values to display; move from **Available** to **Selected**.
- **Secondary Value** ג€” Second value shown as a tooltip on mouseover.
- **Link** ג€” Page link appearing as a secondary value.
- **Open as Pop-up** ג€” Whether the linked page opens as a pop-up.
- **Show Legend Column** ג€” Displays the field label next to the value.
- **Show Column Label** ג€” Displays the column label at the top.
7. Complete the **Time Scale** section:
- **Start Date** ג€” Start date for the time period.
- **Time Scale** ג€” Length of time covered (Specific or Other).
- **Number of Time Periods** ג€” Number of periods to report.
- **Time Period Offset** ג€” Shifts the start relative to the Start Date.
- **Show Group Row Header** ג€” Displays a timescale above the virtual column.
8. In the **Display** section, set **Decimal Places** (enter `0` for integers).
9. Save your changes.

#### Edit a Time Scaled Value

**Using Configure (persists across sessions for that user):**
1. Open **Home**, and from **Resource Management**, click **Resources**.
2. Click the name of a resource.
3. Click **Allocations, Detail**.
4. Open the **Options** menu on the portlet toolbar and click **Configure**.
5. Open the **List Column Section** menu and click **Fields**.
6. Click the **Properties** icon of the Time Scaled Value and edit the values.
7. Save the changes.

**Using Time-scaled Value (session only):**
1. Open **Home**, and from **Resource Management**, click **Resources**.
2. Click the name of a resource.
3. Click **Allocations, Detail**.
4. Open the **Options** menu and click **Time-scaled Value**.
5. Edit the values and save the changes.

#### Add an Aggregation Row for a Number Field

Aggregation rows show statistical data (maximum, minimum, average, sum, count, standard deviation, variance) for money or number fields.

**Steps:**
1. Click the **Options** icon on the portlet toolbar.
2. Open the **Options** menu and click **Configure**.
3. Open the **List Column Section** menu and click **Aggregation**.
4. Click **Add**.
5. Complete the fields:
- **Label** ג€” Name for the aggregation row.
- **Show** ג€” Whether the row name displays.
- **Attribute** ג€” Field value used for the aggregation.
- **Function** ג€” Mathematical function to calculate values.
6. Save your changes.

#### Display an Aggregation Field as a Bar or Column Graph

**Steps:**
1. Click the **Options** icon on the portlet toolbar.
2. Open the **Options** menu and click **Configure**.
3. Open the **List Column Section** menu and click **Aggregation**.
4. Click the word in the **Display** row below the column you want to change.
5. Select the display type and click **Save and Return**.
6. Click **Bar Graph** or **Column Graph** in the Display Row to complete additional setup.
7. Complete the **Primary Column** section:
- **Maximum Length** ג€” Maximum length for the graph in pixels.
- **Length Scaling** ג€” Basis for scaling.
- **Threshold Line Attribute** ג€” Field used to calculate the threshold.
- **Threshold Aggregation Function** ג€” Mathematical function for the threshold.
- **Over-threshold Color** ג€” Color indicating the threshold has been exceeded.
8. Complete the **Stacked Column** section to add a second value:
- **Stacked Attribute** ג€” Second field value displayed as a stacked bar or column graph.
9. Save your changes.

## Configure a Chart Portlet

Chart portlets visually represent complex information. You can:
- Mouse over data points to view dimension values and metrics.
- Drill down to view details about a data point.
- Slice pie or funnel charts.
- Assign colors and change axis labels.

### Configure the Display Options for a Chart Portlet

**Steps:**
1. Do one of the following:
- Go to **Administration > Studio > Portlets** and open a portlet.
- In the portlet title bar, click **Options** and select **Configure**.
2. To resize the chart, click the **General** tab and adjust the **Base Size** field.
3. Click the **Chart Section** menu and select **Options**.
4. Complete the relevant fields (available fields vary by chart type):

| Field | Description |
|---|---|
| **Allow Configuration** | Users can modify portlet appearance |
| **Allow Label Configuration** | Users can modify portlet labels |
| **Angle of First Slice** | Position of the first dividing line (Pie/Funnel charts), 0ג€“360ֲ° |
| **Axis Label** | Displays metric name along an axis |
| **Category Labels** | Labels along the category axis |
| **Datapoint Labels** | Data name next to values on the chart |
| **Decimal Places** | Number of decimal places to display |
| **Display Units** | Controls how values are rounded |
| **Filter** | How results appear initially |
| **Major Unit Increment** | Interval of major ticks on the axis |
| **Maximum Value** | Greatest value to display on the axis |
| **Mouseover Labels** | Values shown on cursor hover |
| **Other Category Threshold** | Groups low-value records into an "Other" category |
| **Show Axis** | Whether the axis line displays |
| **Show Legend** | Whether a chart legend displays (default: on) |
| **Show Line Markers** | Shows data points on line charts |
| **Show Lines** | Connects data points on line charts |
| **Show Major Grid Lines** | Whether major grid lines display |
| **Show Separator** | Comma-separates values > 999 |
| **Show Title** | Displays the portlet name |
| **Sort Column** | Default sort column (column/line charts) |
| **Sub-type** | Shows metrics as separate bars (bar/column charts) |
| **Tick Label Angle** | Angle of labels used with tick marks |

5. (Optional) Select **Guides** in the **Options** field and click **New** to configure a line chart guide:
- **Axis** ג€” Axis for the guide.
- **Label** ג€” Name for the axis.
- **Show Label** ג€” Whether the label displays.
- **Type** ג€” Source of guide information (Attribute, fixed value, or percent).
- **Color** ג€” Guide color (default: Black).
6. Save your changes.

### Determine the Chart Portlet Source Data

**Steps:**
1. Go to **Administration > Studio > Portlets**.
2. Filter by **Type: Filter** to find the portlet.
3. Open the filter portlet and from the **Chart Section** menu, click **Source Data**.
4. Select the metrics to include.
5. Save your changes.

### Configure for Consistent Chart Colors

> [!NOTE]
> The color consistency feature does not apply to heat map and tree map charts.

If your administrator has enabled **Use Consistent Chart Colors** at the system level, you can configure a chart portlet to use consistent colors and a consistent color key.

**Steps:**
1. Open the page containing the chart.
2. Open the **Options** menu and click **Configure**.
3. Complete the fields:
- **Consistent Color Key** ג€” Specifies the color key (available keys depend on the chart's source data).
- **Use Consistent Colors** ג€” Select **Yes** or **No** to override the system-level setting for this chart.
4. Save your changes.

### Apply or Remove Consistent Chart Colors

You can toggle between **Consistent Colors** and **Default Colors** at the chart level.

> [!NOTE]
> Changes made at the chart portlet level during your session are lost when you log out. To keep color changes permanently, configure the chart portlet.

**Steps:**
1. Open a page containing the chart.
2. Select **Consistent Colors** or **Default Colors** from the **Options** toolbar for the chart portlet (the option shown depends on the current setting).

## Configure a Filter Portlet

A filter portlet coordinates filtering across multiple portlets. You can configure it to share filter values at two levels:

- **Page Level** ג€” Filter values are not shared across pages.
- **Application Level** ג€” Filter values are shared across pages (add the same filter portlet to each page).

### Add a Field to a Filter Portlet

**Steps:**
1. Go to **Administration > Studio > Portlets**.
2. Filter by **Type: Filter** and search for the portlet.
3. Open the filter portlet and click **Fields**.
4. Click **Add**.
5. Complete the fields (vary by **Data Type**):
- **Data Type** ג€” Data type for the field (cannot be changed after saving).
- **Show as Percent** ג€” Whether the value displays as a percent.
- **Filter Default** ג€” Default value in the filter field.
- **Width** ג€” Field width (default: 30 px; date fields: 20 px).
- **Required in Filter** ג€” Requires a value when a filter is executed.
- **Hidden in Filter** ג€” Field is hidden at runtime but its default value is still applied.
- **Read-Only in Filter** ג€” Field displays with a default value that cannot be edited.
- **Hint** ג€” Short help message for the user (limit: 512 characters).
6. Save your changes.

Add a Lookup or Multi-Valued Lookup Field to a Filter Portlet

**Steps:**
1. Go to **Administration > Studio > Portlets**.
2. Filter by **Type: Filter** and search for the portlet.
3. Open the filter portlet and click **Fields**.
4. Click **Add**.
5. Complete the fields:
- **Data Type** ג€” Select **Lookup** or **Multi Value Lookup**.
- **Display Type** ג€” How the field is used (**Text Entry** or **Exact Text**).
- **Lookup** ג€” List of values for the user to select.
- **Lookup Style** ג€” How many items a user can select.
- **Entry** / **Exit** ג€” (Static dependent lookups only) Starting and ending points for the lookup data.
- **Filter Default**, **Required in Filter**, **Hidden in Filter**, **Read-Only in Filter**, **Hint** ג€” Same as above.
6. If you selected a parameterized lookup, complete the **Lookup Parameter Mappings** section.
7. Save your changes.
