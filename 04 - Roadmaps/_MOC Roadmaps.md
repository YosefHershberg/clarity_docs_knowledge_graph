# Roadmaps — Map of Content

**Roadmaps** in Clarity PPM support **top-down strategic planning** — allowing leadership and PMOs to plan investments visually on a timeline before detailed project schedules are created. Roadmaps aggregate investments (projects, custom investments, ideas) as bars on a shared timeline, enabling capacity analysis, prioritization, and what-if scenario planning.

> [!tip] Roadmaps vs. Project Timelines
> A **Roadmap** is a high-level strategic view showing many investments over time. A **Project Timeline** (within a single project) shows the detailed task schedule for one project. Use roadmaps for portfolio-level planning and project timelines for execution tracking.

## Key Concepts at a Glance

| Term | Description |
|------|-------------|
| **Roadmap** | A strategic planning canvas showing multiple investments on a shared timeline |
| **Roadmap Item** | A bar on the roadmap representing an investment, idea, or placeholder |
| **Scenario** | A what-if version of the roadmap; compare multiple scenarios without affecting live data |
| **Capacity Bar** | Visual indicator of total resource demand vs. supply across periods |
| **Bar Color** | Can be set manually or driven by a field value (e.g., status, priority) |
| **Top-Down Planning** | Setting investment budgets and dates at the portfolio level before detailed planning begins |
| **Roadmap Blueprint** | Admin template that defines the fields and layout available on a roadmap |
