# Projects — Map of Content

Projects are the primary investment type in Clarity PPM. This section covers the full project lifecycle — from creation through staffing, planning, execution, financial tracking, and reporting.

> [!tip] New to Projects?
> Start with [[Getting Started]] to understand the UI, then [[Create a Project]] to set up your first project. Follow the role path below for a logical learning sequence.

---

## Project Lifecycle Notes

### Foundation
- [[Getting Started]] — Navigating projects: lists, layouts (grid/board/timeline/canvas), filtering, and the project detail view (p984–1001)
- [[Create a Project]] — Creating projects from scratch, copying an existing project, and importing via CSV (p1002–1010)

### Planning & Execution
- [[Staffing]] — Adding team members, replacing/removing staff, mixed bookings, allocation values, and access rights (p1018–1040)
- [[Tasks Phases Milestones]] — Phases, milestones, tasks, and to-dos across grid, board, and timeline layouts (p1041–1075)
- [[Autoschedule]] — Using the Autoschedule engine to let Clarity calculate dates and durations based on dependencies (p1068–1075)
- [[Checklists]] — Creating and managing project checklists to track repeatable quality gates (p1081–1094)

### Documents & Links
- [[Documents and Links]] — Attaching documents and external URLs to a project (p1115–1121)

### Financial Management

### Risk & Issues

### Reporting & Monitoring

## Key Concepts at a Glance

| Term | Description |
|------|-------------|
| **Investment** | Parent category; projects, ideas, and custom investments all share common attributes |
| **Blueprint** | Template that defines which fields and modules are available on a project |
| **Staff OBS** | Organizational breakdown structure used for staffing access |
| **Baseline** | Snapshot of project plan at a point in time for comparison |
| **Autoschedule** | Engine that recalculates task dates based on dependencies and resource calendars |
| **Cost Plan** | Detailed line-item financial plan associated with a project |
| **Canvas** | Fully configurable dashboard view within a project |
