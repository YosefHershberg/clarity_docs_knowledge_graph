# Manage Status Reports

**Status Reports** communicate investment progress to stakeholders. They combine structured data (metrics, RAG indicators) with narrative content and visual widgets into a shareable snapshot of an investment's health.

## Report Components

| Component | Description |
|-----------|-------------|
| **RAG Status** | Red / Amber / Green health indicators for key dimensions (schedule, budget, scope, etc.). |
| **Narrative Sections** | Free-text fields for commentary and context written by the report author. |
| **Key Metrics** | Auto-populated data pulled from the investment (budget, actuals, dates, etc.). |
| **Canvas Widgets** | Visual dashboard elements embedded directly in the report. |

## Status Report Workflow

1. **Create** a draft report on the investment's **Status Reports** tab.
2. **Edit** narrative sections, update RAG indicators, and configure widgets.
3. **Publish** the report to stakeholders. Once published, the report becomes read-only.
4. **Copy** the published report as a template for the next reporting period.

> [!TIP]
> Copying a published report into the next period preserves the structure and widgets, saving setup time for recurring reports.

> [!NOTE]
> Only the **Published** version of a status report is visible to stakeholders without edit access. Draft reports are only visible to the report author and users with the appropriate access rights.
