# Common Capabilities in Blueprints

## Overview
Blueprints define the layout, modules, and behavior for each investment type in Clarity. They control what users see and can do within each workspace.

## Key Capabilities
- **Details Layout** — Configure sections and fields on the details page
- **Module Configuration** — Enable/disable modules (Tasks, Staff, Financials, Risks, etc.)
- **Business Rules** — Conditional logic for field visibility and editability
- **Channels** — Route users to different layouts based on criteria
- **Actions** — Associate processes with blueprints

## Creating a Blueprint
1. Navigate to Administration > Blueprints
2. Select the object type (Project, Idea, etc.)
3. Configure the Details layout
4. Add and configure modules
5. Set up business rules
6. Publish the blueprint

## Business Rules
- Control field visibility based on conditions
- Make fields read-only based on status
- Hide/show modules based on user role
- Enforce data validation
