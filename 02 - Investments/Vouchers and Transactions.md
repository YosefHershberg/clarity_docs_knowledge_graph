# Manage Vouchers and Transactions

## Overview

The **Transactions** workspace represents actual financial activity recorded against investments. It provides financial managers with tools to track, review, and reconcile financial data across the investment portfolio.

## Transaction Types

| Type | Description |
|------|-------------|
| **Labor** | Time-based costs derived from approved timesheet entries |
| **Non-Labor** | Expenses, materials, equipment, and other non-people costs |
| **Adjustments** | WIP adjustments and corrections to previously posted transactions |

---

## Manage Unposted Transactions

**Unposted transactions** are financial entries that have been generated but are awaiting approval or posting to the general ledger:

1. Navigate to the **Transactions** workspace
2. View unposted transactions in the grid
3. Review and validate transaction details (investment, resource, amount, period)
4. Post transactions to finalize them

> [!note]
> Transactions are typically generated when timesheets are approved or when expenses are entered. Posting moves them from WIP (Work in Progress) to actuals.

---

## View Posted Transactions

**Posted transactions** are finalized financial records that have been applied to investment actuals:

- Filter by **investment**, **date range**, or **transaction type**
- View transactions across multiple investments simultaneously
- Use **Column Panel** to add/remove attributes in the grid
- Export to **CSV** for external analysis or reconciliation

---

## Working with the Grid

The Transactions grid supports standard Clarity grid capabilities:

- **Filters** — Narrow results by investment, date, resource, type
- **Group By** — Group transactions by investment, cost category, or other attributes
- **Saved Views** — Save filter and column configurations for reuse
- **Export** — Export up to 250,000 rows to CSV
