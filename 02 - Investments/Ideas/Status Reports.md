# Manage Status Reports for Ideas

## Overview

Status reports for ideas communicate progress and evaluation findings to stakeholders. The module works the same way as for other investment types, with the same Canvas view and Reports Grid interface.

## Required Access Rights

| Action | Access Right |
|--------|-------------|
| Create a new status report | \<Idea\> - Edit and Status Report access right |
| Edit existing status reports | **Status Report - Edit All** and **Project - Edit** |
| View status reports | **Status Report - View All** |

---

## Working with Idea Canvas

The Canvas view provides a visual, drag-and-drop interface for creating status reports:

- Configure the layout with **Text**, **Status**, **Data**, and **Embedded** widgets
- Add **RAG indicators** (Red/Amber/Green) for quick status communication
- Resize, rearrange, and remove widgets as needed
- Modify the canvas by dragging available widgets or fields to the desired position

**Key canvas capabilities:**
- Edit data directly on the canvas
- Export the canvas to **PDF** for stakeholder sharing
- Expand widgets for a full-screen view

---

## Working with Reports Grid

The Reports Grid provides a list-based view of all status reports for the idea:

- View the current and all prior status reports
- Filter by **date range**, **status**, and **author**
- Copy previous reports as templates for new reports
- Use **Bulk Edit** to update multiple rows simultaneously

---

## Publishing Status Reports

1. Create a draft status report (or open an existing draft)
2. Fill in sections and update widgets with current data
3. Click **Preview** to review the formatted PDF
4. Click **Publish** to distribute the final report to stakeholders

> [!note]
> Published reports have **Report Status** set to **Final** and become read-only. To make changes, change the Report Status back to **Draft**, edit the report, then publish again.
