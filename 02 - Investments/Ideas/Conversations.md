# Idea Conversations

## Overview

The **Conversations** module enables team communication within the context of an idea, keeping all discussions tied to the relevant initiative. Team members can start threads, reply, and @mention others — all without leaving the idea record.

## Features

- Post messages visible to all idea team members
- Reply to existing conversation threads
- **@mention** specific team members to notify them
- Attach files and links to messages
- View the full conversation history

## Prerequisites

The Conversations module must be enabled in the Idea blueprint by an administrator. In addition, users need the appropriate access rights.

## Access Rights for Conversation Actions

| Action | Access Right Required |
|--------|-----------------------|
| View a message | View access to the idea |
| Reply to a message | View access to the idea |
| Attach files | View access to the idea |
| Navigate to the idea | Idea Management - Navigate |

## Starting a Conversation

1. Open the relevant idea
2. Navigate to the **Conversations** module (tab or section in the details panel)
3. Type your message in the text field
4. To notify a team member, type **@** followed by their name and select them from the list
5. (Optional) attach a file or link using the attachment icon
6. Click **Post** to submit your message

> [!tip]
> @mentioning a resource manager or project sponsor directly in the idea conversation ensures they are notified and can respond without you needing to track them down separately.

## Replying to a Message

1. Locate the message in the Conversations thread
2. Click **Reply** beneath the message
3. Type your response and click **Post**

## Attaching Files

- Files can be attached directly to a conversation message
- Supported via the attachment icon in the message compose area
- Attachments provide context for reviewers without requiring separate file-sharing tools

## Notifications

When a user **@mentions** you in an idea conversation:

- A notification appears on the **bell icon** at the top of all Clarity pages
- The badge count on the bell shows the number of new notifications
- The notifications panel shows the sender's avatar and the first line of their message
- Click a notification to open the conversation context — you can reply, edit, or delete the message from there

> [!note]
> Notifications are user-specific. Deleting a conversation also deletes its associated notifications for all users.
