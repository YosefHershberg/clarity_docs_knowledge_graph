# Getting Started with Team Investments

## Overview

**Team Investments** represent persistent, long-term funded teams organized around products, services, and value streams — rather than temporary project-based efforts. Instead of funding individual projects, organizations fund teams that deliver continuous value.

This model supports **Agile at scale** and **product-led operating models**, where teams are stable units that own a product or capability long-term.

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Persistent Funding** | Teams receive ongoing operational budgets rather than time-bounded project budgets |
| **Value Stream Alignment** | Teams are organized around products, services, or business capabilities |
| **Continuous Delivery** | Focus on ongoing value delivery rather than project completion |
| **Stable Composition** | Team membership changes less frequently than project staffing |

| Access Right | Description |
|-------------|-------------|

## Creating Team Investments

### From Scratch

1. Navigate to the **Team Investments** workspace
2. Click **+** to create a new team investment
3. Fill in team **name**, **description**, and required attributes
4. Click **Save**

### From Template

1. Click **+ New from Template**
2. Select a team investment template
3. The new investment inherits the template's blueprint configuration, task structure, and financial setup

### CSV Import (Beta)

1. Click **Import** in the toolbar
2. Download the CSV template for team investments
3. Populate the template with team data
4. Upload and map columns to Clarity fields

## Navigating Layouts

Team Investments support multiple layouts:

| Layout | Best For |
|--------|----------|
| **Tiles** | Visual overview of all teams with status indicators |
| **[[Grids\|Grid]]** | Detailed data management across all team attributes |
| **[[Board Layout\|Board]]** | Kanban-style drag-and-drop status management |

## Blueprint Configuration

The Team Investment blueprint must be configured by an administrator before use:
- Add relevant sections and fields to the **Details** layout
- Enable modules: **Tasks**, **Staff**, **Financials**, **Risks**, **Issues**, **Changes**, **Canvas**
- Publish the blueprint and set it as default

> [!important]
> Without a published blueprint, team investments cannot be created or used.
