# Getting Started with Classic PPM

> [!note]
> Classic PPM is the legacy interface for Clarity. While Clarity (Modern UX) is the primary interface, Classic PPM provides access to advanced configuration and features not yet available in the modern UI.

## Overview

Classic PPM provides a traditional web application interface for managing projects, resources, and portfolios. It includes features like:

- **Dashboards and Portlets** — Configurable dashboard pages with data portlets
- **Gantt Charts** — Traditional project schedule visualization
- **Portfolio Management** — Investment analysis and optimization
- **Process Management** — Workflow processes and automations
- **Administration** — Full system configuration interface

## Accessing Classic PPM

1. Log in to Clarity
2. Click the **Classic PPM** link (or navigate to the Classic PPM URL directly)
3. The Classic PPM home page displays your configured dashboards

## Key Differences from Clarity (Modern UX)

| Feature | Classic PPM | Clarity (Modern UX) |
|---------|------------|-------------------|
| **Navigation** | Tab-based pages | Side menu with workspaces |
| **Layout** | Portlet-based dashboards | Grids, boards, timelines |
| **Project Views** | Gantt, list, properties | Grid, board, timeline, tiles |
| **Editing** | Page-by-page forms | Inline editing, details panel |
| **Mobile** | Limited | Full mobile app |

## When to Use Classic PPM

Classic PPM is still needed for:
- [[Processes Configuration|Process configuration]] and management
- [[Portfolio Management|Portfolio analysis]] and optimization
- Advanced [[_MOC Administration|administration]] tasks
